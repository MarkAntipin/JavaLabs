Билет №9

1) Исключения. Иерархия исключений.

	Исключение — это ошибка возникающая во время выполнения программы.

	Все исключения в Java являются объектами, следовательно их можно вызывать
	специально, когда требуется сообщить пользователю о критической ситуации и
	завершить программу

	Базовый класс исключений - Throwable, от него наследуюстся Exception и Error

	Дерево:
	- Throwable
		- Exception - это проблема в программе, которая может быть решена
			- RuntimeException
			- IOException
			- SQLException
		- Error - серьезные проблемы, которые не следует обрабатывать, связаны с ошибками на уровне Java машины
			- VMError

2) Какие методы называются перегруженными?
   
   В Java методы называются перегруженными если они внутри одного класса имеют одинаковое имя, но объявление их параметров различны (один из способов реализации полиморфизма)

3) String, StringBuffer и StringBuilder

   String - этот класс представляет собой последовательность символов
   - immutable - нельзя менять
   - final -  не может быть наследников
   (применять при работе со строками, которые редко будут модифицироваться)

   StringBuffer - класс для модификации String без создания мусора
   - mutable - можно менять
   - потокобезопасный - его экземпляры могут использоваться в нескольких потоках
   (применять при работе со строками, которые часто будут модифицироваться в многопоточной среде)

   StringBuilder - класс как и StringBuffer но потокобезопасный
   - потоконебезопасный - работает быстрее благодаря несинхронизированности методов
   (применять при работе со строками, которые часто будут модифицироваться, в однопоточной среде)
